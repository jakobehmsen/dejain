Welcome to the project page of Dejain

Dejain is a declarative Java bytecode instrumentation language with a syntax leaning towards Java. Its imagined applications lie within model driven development by using POJOs as models.

Take a look below to see how easy it is to add an extra field and override the toString method of all classes being loaded:

class {
    +private String extraField = "Hello World!";

    +public String toString() {
        return "extraField=" + extraField;
    }
}

Code interpolation

There are limitations to instrumentation based only on code ad verbatim. To overcome this, Dejain supports interpolation of explicitly generated code. This code is represented by ASTs, where these ASTs may be implictly converted from non-AST values (e.g. Strings and ints) into their corresponding AST represention.

In the below example, code interpolation is shown where the class name of the class being transformed is explicitly used. This String value is then implicitly converted into a String AST and followingly interpolated into the return statement:

c = class {
    +public String getClassName() {
        return $c.name;
    }
}
